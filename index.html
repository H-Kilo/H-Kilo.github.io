【动漫】全集数据评估：针对漫剧（如《凡人修仙传》动画）全集数据，评估人物识别一致性、台词说话人错误率、场次与情节划分准确性等。

【长剧】《好东西》、《爱情神话》电影效果评估：系统性评估模型在电影长视频理解上的表现，包括台词说话人识别、场次/情节/镜头划分准确度、人物识别错误等维度。

【单集剧情简介】模型测评：测试deepseekV3.1等不同模型，使用镜头、情节、场次三个维度的数据生成单集剧情简介，评估其内容准确性、完整性与人物名称准确度。

代表性案例：动漫底层数据分析模型效果评估

【背景】针对动漫（如《大主宰》）底层数据分析，需要对多个大模型版本（Gemini 2.5pro、3.0等）生成的结构化数据进行系统性评估，验证模型对台词、场次、镜头、情节等维度的理解准确性。评估的核心目标是比较不同模型版本在同一批动漫内容上的表现差异，为模型选型和迭代提供数据支撑。

【过程】

1. 设计评估体系，确定以"预设点位"为基准的对比评估方法，在动漫内容中预先标注关键场景、人物和情节节点作为标准答案；

2. 对每个模型版本输出的底层数据进行逐项比对，包括台词与说话人匹配度、场次边界划分合理性、镜头切换点准确性、情节段落完整性等维度；

3. 统计各模型版本在各项指标上的通过率，将错误类型归纳为"人物识别错误"、"场次漏分/错分"、"台词归属错误"等类别，量化各类问题的占比；

4. 输出对比评估报告，明确各模型的优劣势和适用场景建议。

【成果】输出了覆盖4个模型版本、3部动漫的横向对比评估报告，系统量化了各模型在动漫内容理解上的差异，其中Gemini 3.0在人物识别一致性上较2.5pro提升明显，但在长对话场景的说话人归属上仍有较高错误率。评估结果被用于指导后续模型版本的选择和优化方向。

02 问题排查与优化

【检索】Query检索不准确下钻排查：系统分析片段库检索失败的原因，将错误分类为"底层数据理解问题"、"检索策略问题"和"向量检索失败归因"，并量化了"只识别人物，未识别其他信息"等具体问题的占比。

【匹配】解说词匹配画面不准确排查：定期根据业务反馈（MCN-缪斯），排查AI生成解说词与实际匹配画面不符的问题，定位问题来源（如底层数据错误、检索词不合理等）并提出优化方向。

【数据】未上线介质内容理解问题排查：针对未播或正在播的内容（未上线介质）使用独立模型导致的内容理解偏差进行专项排查，发现主要问题是人物识别不准/不完全以及画面/剧情理解能力弱。

【质量】分镜脚本剪跳帧问题定位：排查"分镜脚本剪"工具输出的视频出现跳帧的问题，通过分析工作流日志，定位到问题根源是原始视频切割本身出了错误，而非大模型后期编造。

代表性案例：Query检索不准确下钻排查

【背景】在AI解说视频生成流程中，系统根据解说词生成检索Query，从片段库中匹配对应画面。业务方反馈部分解说词匹配到的画面与内容不相关或质量不佳，需要系统性地排查检索失败的根本原因，以提升画面匹配的准确率。

【过程】

1. 收集近期检索失败的案例样本，覆盖不同类型的内容（动漫、长剧、电影）和不同的解说风格；

2. 将检索失败案例逐条下钻分析，追溯到Query生成、向量检索、底层数据三个环节，将错误类型分为三类：底层数据理解问题（模型对内容理解错误，如人物、场景识别错误）、检索策略问题（Query关键词选取不当或权重设置不合理）、向量检索失败（语义匹配偏差）；

3. 量化各类问题的占比，其中"底层数据只识别人物，未识别场景/动作/情绪等其他信息"这一细分类别占比最高，是导致检索失败的主要原因；

4. 针对每类问题提出了对应的优化建议，包括优化底层数据的标注维度、改进Query生成策略、调整向量检索的相似度阈值等。

【成果】输出了详细的检索失败归因分析报告，将零散的"匹配不准"反馈归纳为系统性的问题分类体系。报告被产品和技术团队采纳，"底层数据增加非人物维度标注"的建议被纳入后续模型迭代计划，后续检索准确率有显著提升。

03 工作流（dify）搭建与功能建设

【功能增加】为多个解说工作流增加"拆分分镜节点"：在"剧情向人物独白解说"、"爆文衍生解说"等工作流中，增加了自动将长解说稿拆分为分镜头脚本和画面描述的节点，并优化了相关Prompt，使画面描述更适合作为检索Query。

【功能优化】"爆文衍生"解说工作流优化：优化了解说稿生成的Prompt，使其更紧扣爆文模板，并增加了"单集ID"和"核心角色"等输入字段，以提升画面匹配的精准度。

【工具集成】新增"人工介入"节点：在"混剪转解说"等工作流中添加了人工介入节点，并为其适配了后续的"生成画面描述和字幕内容"流程，解决了人工调整分镜后无法自动生成后续内容的问题。

【流程打通】未上线介质匹配增加"目标集"限制：修改了未上线介质相关的工作流，确保解说词匹配的画面必须来自指定的剧集ID（目标集），解决了跨剧集错误匹配的问题。

【新工具测试】Agent三合一工具测评：测试了集成"意图分析+片段检索+花字生成"的新版Agent工具，重点评估了其利用人物关系图谱进行角色识别的准确性。

代表性案例：【混剪转解说】工作流适配与优化

【背景】"混剪转解说"是面向MCN机构（缪斯）的核心工作流之一，用于将已有的混剪视频自动转化为带有AI解说的完整视频。业务方反馈工作流在以下场景存在问题：

1）人工调整分镜后无法自动生成后续的解说内容；

2）生成的画面描述与检索匹配度不高，导致画面与解说不同步；

3）缺少对未上线介质内容的画面匹配限制，容易出现跨剧集错误匹配。

【过程】

1. 分析现有工作流的节点链路，识别断裂点——人工介入节点之后缺少自动衔接"生成画面描述和字幕内容"的流程；

2. 新增"人工介入"节点并适配后续流程，确保运营人员手动调整分镜后，系统能自动基于调整结果继续生成画面描述和字幕；

3. 优化画面描述生成的Prompt，使其产出更适合作为检索Query的描述文本（增加场景、动作、情绪等维度的描述）；

4. 为未上线介质工作流增加"目标集"限制参数，强制检索范围限定在指定剧集ID内；

5. 增加"拆分分镜节点"，将长解说稿自动拆分并生成对应的画面描述，减少人工拆分的工作量。

【成果】优化后的工作流实现了"混剪→人工介入→自动生成解说内容"的完整闭环，运营人员调整分镜后无需手动补写后续内容。画面匹配精准度明显提升，"目标集"限制彻底解决了跨剧集错误匹配的问题。该工作流后续成为缪斯平台解说生产的标准流程之一。

代表性案例：撰写PRD文档《解说字幕擦除OCR识别准确度提升》

【背景】业务方反馈"分镜脚本剪"工具在进行OCR识别时准确度不足，具体表现为：

1）字幕区域识别不完整，部分字幕行被遗漏；

2）字幕擦除后残留痕迹影响后续OCR识别；

3）特殊字体（艺术字、手写体）的识别准确率明显低于常规字体。该问题直接影响了解说视频的自动生成质量，需要从产品层面系统性地推动优化。

【过程】

1. 收集具体的问题案例，与业务方确认问题的表现形式、频率和影响范围；

2. 撰写PRD文档，结构包括：问题定义（明确当前OCR识别在字幕擦除场景下的不足）、目标（量化准确度提升指标，如常规字体识别准确率≥98%、特殊字体≥90%）、需求描述（分功能需求和非功能需求详细阐述）、验收标准（定义各场景下的测试用例和通过条件）；

3. 与算法和工程团队评审PRD，确保技术方案的可行性，并根据反馈调整需求细节；

4. 推动PRD进入开发排期，并持续跟进开发进度和测试结果。

【成果】PRD文档获得产品和技术团队的认可，需求被纳入迭代计划。文档中提出的"分字体类型差异化处理"和"字幕擦除与OCR分离为独立模块"两个核心建议被采纳，为后续OCR识别准确度的系统性提升提供了清晰的产品方案和验收标准。
